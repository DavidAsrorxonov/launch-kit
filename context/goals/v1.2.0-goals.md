# LaunchKit v1.2.0 Goals

## Objective

Introduce a fully functional authentication module so users can generate Next.js starter projects with:

- Better Auth
- Google OAuth
- Protected routes
- Automatic MongoDB integration
- Complete environment variable setup

The goal of v1.2.0 is to allow developers to generate a real, production-ready authentication system with a single click.

---

## Primary Goals

### 1. Add Better Auth + Google OAuth Module

Users should be able to select:

- No Authentication
- Better Auth + Google OAuth

When selected, LaunchKit should generate all required authentication files and dependencies.

---

### 2. Automatically Include MongoDB

Because Better Auth requires a database, selecting authentication should automatically:

- Enable MongoDB
- Disable the "No database" option
- Show MongoDB in the live summary

This prevents invalid project configurations.

---

### 3. Generate Authentication Files

The generated project should include:

```txt
app/login/page.tsx
app/dashboard/page.tsx
app/api/auth/[...all]/route.ts
lib/auth/auth.ts
lib/auth/auth-client.ts
```
