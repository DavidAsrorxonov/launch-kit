# Task: Fix Generated README Formatting and Environment Setup Notice

The current application stage version is `v1.2.0`.

Refer to `context/goals/v1.2.0-goals.md` for the latest project goals and scope.

Everything specified in the v1.2.0 goals is currently working. However, there are issues with the `README.md` file generated inside downloaded projects.

## Problem

When a user generates a project, downloads the ZIP file, unzips it, and opens the generated `README.md`, the Markdown formatting appears broken.

The content is being rendered as one continuous line instead of properly formatted Markdown.

Example of the current broken output:

# project Generated by LaunchKit 🚀 ## Requirements - Node.js 20.9 or newer - npm, pnpm, yarn, or bun ## Run locally ```bash npm install npm run dev

This makes the generated `README.md` difficult to read and unprofessional.

## Required Fixes

### 1. Fix Generated README Formatting

Update the generated project README template so that the output Markdown is clean and properly structured.

The generated `README.md` should contain correct spacing between:

- Title
- Description
- Requirements
- Environment setup
- Installation steps
- Local development commands
- Optional MongoDB notes
- Optional Better Auth notes

Most likely file to update:

templates/base/README.md.ejs

If the README content is assembled from module partials, also check related module templates.

### 2. Add `.env.example` Setup Instructions

The generated `README.md` must clearly instruct users to copy or rename `.env.example` to `.env`.

Include this command:

```bash
cp .env.example .env
```
